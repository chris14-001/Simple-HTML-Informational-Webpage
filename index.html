Introduction

HTML Basics and Things to Know

HyperText Markup Language — HTML — is the foundational language of the web. Every webpage you have ever visited is built on it. Unlike programming languages that execute logic, HTML is a markup language: it describes the structure and meaning of content by wrapping it in tags. A browser reads those tags and renders them as the headings, paragraphs, images, forms, and links you see on screen.

Learning HTML is not simply about memorising tags. It is about understanding why each element exists, what role it plays in a document, and how browsers, search engines, and assistive technologies interpret it. A page can be made to look correct while being structurally broken — and those breakdowns surface as accessibility failures, SEO penalties, and maintenance problems down the line.

This guide covers the core concepts every developer should have a firm grip on: document structure, semantic markup, attributes and linking, images and media, forms, and accessibility. Each section highlights not just how something works, but the decisions and habits that separate solid HTML from code that merely functions.

Attributes and Links

This small piece of code shows how a basic link should work using HTML. 


        <!-- External link, opens in new tab -->
<a href="https://google.com" target="_blank">Google</a>
 
<!-- Internal link (same site) -->
<a href="/about.html">About</a>
 
<!-- Anchor link (jumps to an id on the page) -->
<a href="#contact">Go to contact section</a>
 
<!-- Email link -->
<a href="mailto:hello@example.com">Email me</a>

Images and Media

Every <img> element must include an alt attribute — this is non-negotiable. For informative images, the alt text should describe the content and function of the image. For decorative images that add no meaningful information, use an empty alt attribute (alt="") so screen readers skip over it entirely. Missing alt attributes fail accessibility audits and are a common source of poor search engine indexing.

Image dimensions matter for performance and layout stability. Always specify width and height attributes on <img> elements so the browser can reserve the correct space before the image loads, preventing cumulative layout shift — a Core Web Vital metric directly tied to user experience and SEO rankings. For responsive images, use CSS (max-width: 100%; height: auto;) rather than omitting dimensions entirely.

For modern web work, the <picture> element and the srcset attribute on <img> allow you to serve different image formats or resolutions based on screen size and device capability. Serving WebP to supporting browsers while falling back to JPEG for older ones is a pattern every developer should be comfortable with, as it significantly reduces page weight without any visual quality loss for most users.


Forms and Inputs

HTML inputs are the primary mechanism through which users submit data. Every input must be associated with a <label> — either by wrapping the input inside the label, or by matching the label's for attribute to the input's id. Placeholder text is not a substitute for a visible label; it disappears on focus and provides no persistent context, making forms harder to complete and inaccessible to many users.

Choosing the correct type attribute on <input> elements is more important than many developers realise. Using type="email", type="tel", or type="number" triggers the appropriate keyboard on mobile devices, invokes native browser validation, and signals intent clearly to assistive technologies. Defaulting to type="text" for everything is a missed opportunity that degrades the mobile experience.

Form validation should be layered: use HTML5 constraint attributes (required, minlength, pattern) for immediate browser-level feedback, then supplement with JavaScript for more complex rules, and always validate server-side as the authoritative check. Never rely solely on client-side validation for security — any client-side constraint can be bypassed. The autocomplete attribute is also worth setting deliberately; it controls how browsers autofill fields and significantly speeds up form completion for returning users.

Accessibility Fundamentals

Accessibility is not a feature to be added at the end — it is a quality dimension baked into correct HTML from the start. The single most impactful habit is using the right HTML element for the right job. A <button> element is keyboard focusable, fires on Enter and Space, and announces itself as a button to screen readers automatically. A <div> styled to look like a button does none of those things unless you manually re-implement every behaviour — a common and unnecessary source of accessibility failures.
